# MemoryPool

Система управления памятью на C++

## Описание

Этот проект реализует пул памяти, который позволяет эффективно управлять выделением и освобождением памяти в C++. Пул памяти помогает избежать фрагментации и улучшает производительность при частом выделении и освобождении небольших блоков памяти.

## Файлы

- `IMemoryPool.h`: Интерфейс для пула памяти, определяющий основные методы для работы с памятью.
- `MemoryPool.h`: Заголовочный файл для реализации пула памяти.
- `MemoryPool.cpp`: Реализация методов пула памяти.
- `main.cpp`: Пример использования пула памяти.

## Компиляция

Для компиляции проекта используйте команду:

```bash
g++ -o MemoryPool src/*.cpp
```

## Использование

Запустите скомпилированный файл:
```bash
./MemoryPool
```

## Пример работы
При запуске программы будет создан пул памяти размером 1024 байта. Программа выделяет два блока памяти (32 и 64 байта), а затем освобождает их. В процессе работы выводится информация о выделенной и освобожденной памяти.

Ожидаемый вывод:

1. Пул памяти размером 1024 байт создан.
2. Успешно выделено 32 байта для block1.
3. Успешно выделено 64 байта для block2.
4. Успешно освобожден block1.
5. Успешно освобожден block2.
6. Используемая память: 0 байт.
7. Общий размер пула: 1024 байт.
8. Пул памяти уничтожен.

## Возможности
- Выделение памяти заданного размера.
- Освобождение ранее выделенной памяти.
- Получение информации о текущем использовании памяти и размере пула.

## Исключения
Программа обрабатывает следующие исключения:

- std::invalid_argument: Выбрасывается при попытке выделить нулевой блок памяти или освободить нулевой указатель.
- std::bad_alloc: Выбрасывается при недостатке памяти в пуле.
- std::runtime_error: Выбрасывается при попытке двойного освобождения блока памяти.
